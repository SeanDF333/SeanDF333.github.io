'Inverse' is All you Need

Abstract

Inverse Reinforcement Learning (IRL) enables agents to infer reward functions by observing expert behavior, addressing the challenges of manual reward design and bias in traditional Reinforcement Learning (RL).

While classical IRL methods, such as linear programming IRL and Apprenticeship Learning, laid the groundwork, they often suffer from multiple valid reward functions and limited generalization.Cooperative IRL (CIRL) reframes IRL as a human–agent cooperative game, allowing the agent to learn the true reward through iterative interaction with a human who knows the reward.

Recently, IRL has been extended to fine‑tune Large Language Models (LLMs) by extracting sequence‑level rewards via inverse soft Q‑learning, offering improvements in generation diversity and alignment over token‑level maximum likelihood estimation (MLE). Despite these advances, open questions remain in scalability, reward identifiability, and robustness when applying IRL and CIRL to complex, high‑dimensional LLM settings.

1. Introduction : Why We Need “I”

Inverse Reinforcement Learning tackles the inverse problem of standard RL by inferring the underlying reward function from observed expert behavior rather than optimizing a predefined reward.

In LLMs, IRL offers the potential to extract human‑aligned sequence‑level rewards, moving beyond token‑level supervision (MLE) and thus improving generation coherence and adherence to human preferences.

However, classical IRL assumes a single agent and known dynamics, which can misalign with human intent; Cooperative Inverse Reinforcement Learning (CIRL) addresses this by modeling human and agent as collaborators maximizing a shared but partially known reward, thus enabling more robust human–LLM alignment.

2. Classical IRL

2.1 Reinforcement Learning vs. Inverse Reinforcement Learning

In Reinforcement Learning, the agent learns to maximize a predefined reward function through interactions with the environment. The environment is typically modeled as a Markov Decision Process (MDP):

The objective is to find a policy that maximizes the expected cumulative reward:

$$
J(\pi) = \mathbb{E}\pi\left[\sum{t=0}^\infty \gamma^t r(s_t, a_t)\right]
$$

In contrast, Inverse Reinforcement Learning seeks to infer the underlying reward function r given observed expert behavior, typically in the form of state-action trajectories. The goal is to find a reward function under which the expert’s policy is optimal.

2.2.2 Apprenticeship learning (Abbeel, P., & Ng, A. Y.2004)

  1. Initialize

    • Select an arbitrary policy (\pi^{(0)}).
    • Compute its feature expectations (\mu^{(0)} = \mathbb{E}{\pi^{(0)}}[\sum{t=0}^\infty \gamma^t \phi(s_t)]).
    • Set iteration counter (i \leftarrow 1).

  2. Inverse-RL Step (Max-Margin Reward Inference)
    Solve the quadratic program:
    $$
    w^{(i)},, t^{(i)} = \arg\max_{|w|2 \le 1};\min{0 \le j < i}; w^\top\bigl(\hat\mu_E - \mu^{(j)}\bigr),
    $$
    where (\hat\mu_E) is the empirical expert feature expectation and (\mu^{(j)}) are past policy expectations.

  3. Convergence Check

    • If (t^{(i)} \le \varepsilon), terminate: the current set of policies matches the expert within tolerance (\varepsilon).

  4. Policy Improvement via RL

    • Define learned reward (R^{(i)}(s) = (w^{(i)})^\top \phi(s)).
    • Run a standard RL solver on MDP(\setminus R) augmented with (R^{(i)}) to obtain new policy (\pi^{(i)}).

  5. Update Feature Expectations

    • Compute (\mu^{(i)} = \mathbb{E}{\pi^{(i)}}[\sum{t=0}^\infty \gamma^t \phi(s_t)]).

  6. Iterate

    • Increment (i \leftarrow i + 1) and return to step 2.

Advantages and Limitations

Advantages

  • Automatic Reward Discovery
    Learns a reward function directly from expert demonstrations, removing the need for manual reward engineering.

  • Performance Guarantees
    Provably converges in (O\bigl(\tfrac{k}{(1-\gamma)^2\varepsilon^2}\ln\tfrac{k}{(1-\gamma)\varepsilon}\bigr)) iterations, yielding a policy within (\varepsilon) of the expert’s true performance.

  • Sample Efficiency
    Requires fewer expert trajectories than naive imitation or parameterized policy methods to achieve comparable performance.

Limitations

  • Quadratic Programming Overhead
    Each inverse-RL step relies on a QP (or SVM) solver, which can be computationally intensive for high-dimensional feature spaces.

  • Feature Engineering Dependence
    Quality of learned policy hinges on the choice of features (\phi(s)); poor features lead to suboptimal behavior.

  • Scalability Challenges
    As the number of features (k) or candidate policies grows, the optimization problem size increases, impacting runtime.

  • Coverage Requirements
    Performance degrades if expert demonstrations do not adequately cover critical regions of the state space.

4. IRL for LLMs

4.1 对抗逆强化学习 (Adversarial IRL)

Fu et al. (2017) 将 GAN 思想引入 IRL，通过生成网络生成策略、判别网络区分专家/策略轨迹，优化目标为
[
\min_\pi \max_D ; \mathbb{E}{τ\sim\pi_E}[\log D(τ)] + \mathbb{E}{τ\sim\pi}[\log(1 - D(τ))]
]
有效提升了在复杂环境下的学习效率与鲁棒性 :contentReference[oaicite:5]{index=5}。

4.2 贝叶斯 IRL

Ramachandran & Amir (2007) 将奖励函数视为随机变量，利用贝叶斯推断计算后验分布：
[
P(R|D) \propto P(D|R),P(R)
]
其中 (P(D|R)) 表示在给定奖励下生成专家示范的概率，增强了对示范噪声的鲁棒性 :contentReference[oaicite:6]{index=6}。

4.3 深度成本学习 (Guided Cost Learning)

Finn et al. (2016) 以最大熵 IRL 为基础，采用神经网络拟合奖励函数，并结合策略优化，提出目标：
[
\min_θ ; -\sum_{τ \in D_E} \log P(τ|θ) + \lambda \sum_{τ \in D_{π_θ}} \exp\bigl(\sum_t r_θ(s_t)\bigr)
]
显著提升了高维连续控制场景下的性能 :contentReference[oaicite:7]{index=7}。

4.4 相对熵 IRL (Relative Entropy IRL)

Boularias et al. (2011) 通过最小化策略分布与专家分布的相对熵，实现奖励函数学习：
[
\min_R ; D_{\mathrm{KL}}\bigl(P(τ|\pi_E),|,P(τ|\pi_R)\bigr)
]
简化了对最优策略求解的依赖，提高了样本效率 :contentReference[oaicite:8]{index=8}。

5. Future Outlook

I​mitating Language via Scalable IRL (2024) 将 IRL 应用于大语言模型的微调，提出“逆软 Q 学习”（Inverse Soft Q‑Learning）：
[
L_{\mathrm{IRL}} = -\sum_{x,y}\bigl[Q(s,a) - \gamma,V(s’)\bigr] + \alpha,|\theta|^2
]
其中 Q 与 V 由模型参数 θ 定义，并结合时序差分正则化，以序列级别的“未来影响”替代逐 token 的 MLE 优化，从而提高生成连贯性与对齐度 :contentReference[oaicite:9]{index=9}。

6. Reference

逆强化学习自 Ng & Russell (2000) 的 LP 框架，到最大熵 IRL 与现代对抗／贝叶斯方法，再到可扩展 IRL 在 LLM 微调中的应用，已形成从理论到实践的完整谱系。面向大模型的 IRL 可将对齐问题转化为序列级奖励推断，为下一代高质量、可控生成模型提供了新的思路。未来可结合在线／离线混合训练、多源偏好数据融合及不确定性驱动采样机制，进一步提升 IRL 在大规模场景中的可扩展性与稳健性。
