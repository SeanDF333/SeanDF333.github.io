'Inverse' is All you Need

Abstract

Inverse Reinforcement Learning (IRL) enables agents to infer reward functions by observing expert behavior, addressing the challenges of manual reward design and bias in traditional Reinforcement Learning (RL).

While classical IRL methods, such as linear programming IRL and Apprenticeship Learning, laid the groundwork, they often suffer from multiple valid reward functions and limited generalization.Cooperative IRL (CIRL) reframes IRL as a human–agent cooperative game, allowing the agent to learn the true reward through iterative interaction with a human who knows the reward.

Recently, IRL has been extended to fine‑tune Large Language Models (LLMs) by extracting sequence‑level rewards via inverse soft Q‑learning, offering improvements in generation diversity and alignment over token‑level maximum likelihood estimation (MLE). Despite these advances, open questions remain in scalability, reward identifiability, and robustness when applying IRL and CIRL to complex, high‑dimensional LLM settings.

1. Introduction : Why We Need “I”

Inverse Reinforcement Learning tackles the inverse problem of standard RL by inferring the underlying reward function from observed expert behavior rather than optimizing a predefined reward.

In LLMs, IRL offers the potential to extract human‑aligned sequence‑level rewards, moving beyond token‑level supervision (MLE) and thus improving generation coherence and adherence to human preferences.

However, classical IRL assumes a single agent and known dynamics, which can misalign with human intent; Cooperative Inverse Reinforcement Learning (CIRL) addresses this by modeling human and agent as collaborators maximizing a shared but partially known reward, thus enabling more robust human–LLM alignment.

Advantages

  • Automatic Reward Discovery
    Learns a reward function directly from expert demonstrations, removing the need for manual reward engineering.

  • Performance Guarantees
    Provably converges in iterations, yielding a policy within (\varepsilon) of the expert’s true performance.

  • Sample Efficiency
    Requires fewer expert trajectories than naive imitation or parameterized policy methods to achieve comparable performance.

Limitations

  • Quadratic Programming Overhead
    Each inverse-RL step relies on a QP (or SVM) solver, which can be computationally intensive for high-dimensional feature spaces.

  • Feature Engineering Dependence
    Quality of learned policy hinges on the choice of features ; poor features lead to suboptimal behavior.

  • Scalability Challenges
    As the number of features or candidate policies grows, the optimization problem size increases, impacting runtime.

  • Coverage Requirements
    Performance degrades if expert demonstrations do not adequately cover critical regions of the state space.

3. Morden IRL

Cooperative Inverse Reinforcement Learning

3.1 Framework

CIRL formalizes value alignment as a cooperative, partial-information game between two agents:

The human (H): Knows the true reward function but has limited computational capacity
The robot (R): Has greater computational capacity but doesn’t initially know the reward function
Unlike traditional IRL, where the robot passively observes the human’s behavior and then acts independently, CIRL models a setting where both agents actively cooperate to achieve the human’s goals. This cooperative nature fundamentally changes how both agents should behave:

The human is incentivized to provide informative demonstrations that help the robot learn the reward function, even if these demonstrations aren’t optimal for direct task completion
The robot must reason about the information content of the human’s actions and update its beliefs accordingly

3.2 Mathematical Formulation

The CIRL game is formally defined as a tuple (S, {AH, AR}, T, {Θ, R}, P0, γ) where:

S is the set of world states
AH and AR are action sets for the human and robot
T is the state transition function: T(s’ | s, aH, aR)
Θ represents the space of possible reward parameters
R(s, θ) is the reward function parameterized by θ
P0 is the initial state and reward parameter distribution
γ is the discount factor
The key distinction from standard reinforcement learning is that the reward parameter θ is known only to the human. Both agents must act to maximize the expected value with respect to the true (but partially unknown) reward function.

The human’s policy can be represented as πH(aH | s, θ), while the robot’s policy is πR(aR | s, b), where b is the robot’s belief over possible reward parameters.

4. IRL for LLMs

Reframes LLM fine-tuning as an Inverse Reinforcement Learning (IRL) problem, extracting a sequence-level reward rather than relying solely on token-level maximum likelihood estimation (MLE). By reformulating inverse soft Q-learning as a temporal-difference-regularized extension of MLE, the authors establish a principled bridge between MLE and IRL, enabling direct optimization of whole sequences. Empirically, IRL-based fine-tuning achieves improved task performance, higher generation diversity, and more robust reward functions compared to traditional MLE, all on fixed supervised fine-tuning datasets without expensive online sampling. The core algorithm hinges on an additional temporal-difference term that regularizes the MLE objective, balancing complexity with clear benefits in generation quality and alignment.

4.1 Mathematical Formulation

The key insight is captured in the mathematical formulation:

IQLearn = MLE + λ × TD-Regularization

Where TD-Regularization represents temporal difference regularization that helps the model account for the future impact of current token choices, addressing the myopic nature of standard MLE training.

5. Future Outlook

Integrating Inverse Reinforcement Learning (IRL) with Group Relative Policy Optimization (GRPO) holds promise for deriving more human-aligned reward signals while leveraging GRPO’s efficient, critic-free updates.
By first using IRL to infer a structured reward function from expert or preference data, then applying GRPO to optimize policies based on relative group comparisons under r ,
we can combine IRL’s principled reward discovery with GRPO’s stability and memory efficiency.
